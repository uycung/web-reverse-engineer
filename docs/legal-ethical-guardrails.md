# Legal and Ethical Guardrails for Web Reverse Engineering

This document outlines the strict legal and ethical boundary policies governing all deconstruction and reimplementation work in this repository.

## Core Mandate

The projects and scripts in this repository are developed strictly for **educational, audit, and portfolio demonstration purposes**. The goal is **technical deconstruction and clean-room reconstruction**, NOT cloning or copying.

Any work created here must represent a **brand-different implementation** inspired by high-level design principles, rather than a replication of proprietary source material.

---

## Direct Rules

### 1. No Code Copying
* We do **not** copy or reuse proprietary JavaScript bundles, minified source files, custom styles, or shaders from the reference websites.
* All logic, rendering instructions, math, and shaders in our demos must be written from scratch.

### 2. No Asset Piracy
* We do **not** download, host, or link to any copyrighted assets (images, videos, vector graphics, custom fonts, textures, or SVG logos) belonging to the reference brand.
* Demos must utilize placeholder media or custom-generated, royalty-free assets.

### 3. Clear Brand Differentiation
* The branding, copywriting, color palettes, and typographic scales of reference websites must be completely changed.
* Fictional products or neutral branding must be used (e.g., *SignalForge* instead of *Clyde*).
* Do not use reference logos or trademarks in UI demonstrations.

### 4. Semantic and Structural Differentiation
* The structure, grid layout, page flow, and copywriting of sections must be altered. The implementation is a "style transfer" of visual/motion techniques rather than a layout clone.
* The content hierarchy must be designed from scratch to suit the fictional product being showcased.

### 5. Proper Vocabulary & Positioning
* We use professional deconstruction terms:
  * **Allowed Terms:** *“inspired by”*, *“clean-room reconstruction”*, *“style transfer”*, *“technical deconstruction”*, *“brand-different implementation”*.
  * **Forbidden Terms:** *“clone”*, *“copy exactly”*, *“replica”*, *“1:1”*.

---

## Reference Material Policy

Reference websites are treated purely as a black-box system to extract high-level motion principles:
1. **Understand technical mechanisms:** E.g., identifying that a page uses a `<canvas>` element with blending effects.
2. **Abstract principles:** E.g., "slow moving noise overlays overlaying overlapping soft gradient circles".
3. **Write original code:** Implement a distinct visual appearance from scratch.
